# AGENTS.md

## Build & Dev

- **Package manager**: `yarn` (v1), not npm. `yarn.lock` is the lockfile.
- **Dev server**: `yarn dev` (Nuxt dev, port 3000)
- **Production build**: `yarn generate --preset github_pages` → output at `.output/public`
- **No lint/typecheck scripts** in package.json. ESLint config exists but is auto-generated by Nuxt (`.nuxt/eslint.config.mjs`).

## Static Data Pipeline (Clerk + Cloudflare Worker)

`public/blogdata.json` is the **single source of truth** for blog posts.
- Deployed as a static asset on GitHub Pages.
- Read at build-time to prerender pages like `/blog/:slug`.
- Read client-side when rendering blog routes.
- Modified directly via Cloudflare Worker API.

### How Writes Work:
1. Admin edits/creates a post at `/editor/*` (Clerk-authenticated).
2. UI calls Cloudflare Worker endpoint (e.g., `POST/PUT/DELETE /posts`) passing a Clerk session JWT.
3. Cloudflare Worker validates JWT via Clerk JWKS, pulls current `blogdata.json` from GitHub API, modifies the array, and commits the updated file directly to the `main` branch.
4. The push to `main` automatically triggers GitHub Actions (`deploy.yml`) to rebuild and redeploy the site.

## CI Workflows

1. **`deploy.yml`** — Runs on push to `main` (which includes commits made by the Cloudflare Worker). Builds with `yarn generate --preset github_pages` and deploys to GitHub Pages.

## Architecture Notes

- **SSR enabled**, but deployed as static (`yarn generate`). Prerendered routes include homepage, `/blog/:slug` (from `blogdata.json`), and `/sitemap.xml`.
- **Hybrid fallback**: If a post is not in the static `/blogdata.json` cache (e.g. within 1-2 minutes of publishing before CI finishes), blog pages fallback client-side to fetching directly from the public Worker API (`GET /posts?slug=...`). This preserves real-time immediate updates!
- **Editor pages** (`/editor/*`) are SPA-only, not prerendered. They use CKEditor 5 with Clerk Authentication and autosave to localStorage. `robots.txt` excludes these paths.
- **Site URL** for sitemap is hardcoded in `scripts/generateSitemap.js` (check there if deploying to a different domain).
