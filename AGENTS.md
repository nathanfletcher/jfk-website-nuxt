## Static Data Pipeline (Critical)

`public/blogdata.json` is the **cached blog dataset**. It's NOT generated at build time â€” it must exist before build. The Nuxt config (`nuxt.config.ts:52-70`) reads it to prerender blog routes. If it's missing or empty, blog pages won't be generated.

The data is fetched from Strapi via `scripts/fetchBlogData.js`:
- Uses incremental fetching via `.last-fetch-timestamp` (committed to repo)
- Merges by `documentId`
- Retries 3x with exponential backoff
- If the API fails but `blogdata.json` already exists, the script exits 0 and preserves existing data

The CI workflow (`fetch-blogdata.yml`) first **warms up** the Strapi instance (Koyeb free tier cold start) with up to 6 curl pings before running the script.

## Strapi API

Default URL: `https://reliable-bubble-e0aafb3b9e.strapiapp.com/api` (Koyeb-hosted, free tier â€” cold starts). Override via GitHub secret `STRAPI_API_URL` or env var `NUXT_PUBLIC_API_URL`. The secret is used in CI; the env var is used by both frontend and scripts.

## CI Workflows

1. **`fetch-blogdata.yml`** â€” Runs every 6h (schedule), on manual dispatch, or via Strapi webhook (`repository_dispatch: strapi-content-updated`). Steps: checkout â†’ warmup Strapi â†’ fetch blogdata â†’ commit blogdata â†’ generate sitemap â†’ commit sitemap â†’ trigger deploy via `repository_dispatch: trigger-deploy`.

2. **`deploy.yml`** â€” Triggered by push to `main`, manual dispatch, or `repository_dispatch: trigger-deploy`. Builds with `yarn generate --preset github_pages` and deploys to GitHub Pages.

**Important**: The fetch workflow runs `node scripts/fetchBlogData.js` directly (no `yarn install`). It disables `postinstall` (which runs `nuxt prepare`) to avoid needing all deps.

## Architecture Notes

- **SSR enabled**, but deployed as static (`yarn generate`). Prerendered routes include homepage, `/blog/:slug` (from `blogdata.json`), and `/sitemap.xml`.
- **Hybrid fallback**: Blog page components try client-side Strapi fetch if a post isn't in the static cache (see `pages/blog/[id].vue:68`).
- **Editor pages** (`/editor/*`) are SPA-only, not prerendered. They use CKEditor 5 with autosave to localStorage. `robots.txt` excludes these paths.
- **Site URL** for sitemap is hardcoded in `scripts/generateSitemap.js` (check there if deploying to a different domain).
